# オンラインコミュニティは クイズ 🎯

みんなで四択クイズを **作って・解いて・直し合える** Webアプリです。
インストール不要。スマホでもパソコンでも、ブラウザで開くだけで使えます。

## できること

- **Misskeyのアカウントでログイン**して参加
- クイズは **3つの難易度**（ビギナー / 中級 / マニア）に分けて出題
- **問題文と4つの選択肢** を、みんなが自由に作れる
- 「この問題ちょっとおかしいかも」と思ったら **誰でも直せる**
- 各問題に **コメントや補足** を書き込める
- 問題の下に **「誰が作った・誰が直した・いつ更新した」** が残る
- 解き終わったら **Misskeyにワンタップでシェア**

---

## 🚀 使えるようにするまで（初回だけの準備）

準備は **2つ** だけです。むずかしい言葉が出てきますが、**書いてある通りにコピペするだけ** で終わります。所要時間はだいたい10〜15分。

> このアプリは「見た目の部分」と「みんなの問題を保存しておく倉庫」の2つでできています。
> 準備1で **倉庫** を用意し、準備2で **見た目の部分をネットに公開** します。

---

### 準備1：問題をしまっておく「倉庫」を用意する（Supabase）

「Supabase（スーパーベース）」という無料サービスを、問題データの保管庫として使います。

1. **アカウントを作る**
   [supabase.com](https://supabase.com) を開いて、右上の「Start your project」からアカウントを作成します（GitHubアカウントでそのままログインできます）。

2. **プロジェクト（＝あなた専用の倉庫）を作る**
   「New project」を押します。
   - 名前（Name）：`community-quiz` など、好きな名前でOK
   - Database Password：好きなパスワード。**あとで使わないのでメモしなくても大丈夫**ですが、念のため控えても◎
   - Region（場所）：`Northeast Asia (Tokyo)` を選ぶと日本から速いです
   - 「Create new project」を押して、1〜2分待ちます（倉庫の建設中です）

3. **倉庫に「棚」を作る（コピペ作業）**
   - 左側のメニューから **「SQL Editor」** を開きます
   - このアプリのフォルダにある **[`supabase/schema.sql`](supabase/schema.sql)** というファイルを開いて、**中身を全部コピー**します
   - SQL Editorの入力欄に **貼り付けて**、右下の緑の **「Run」** ボタンを押します
   - 「Success」と出れば、問題・コメント・履歴をしまう棚ができました 🎉

4. **倉庫の「住所」と「合いカギ」をコピーする**
   - 左側メニューの一番下 **「Project Settings（歯車マーク）」→「API」** を開きます
   - 次の2つが表示されています。あとで使うのでこの画面は開いたままにしておきます：
     - **Project URL**（住所）：`https://〜〜〜.supabase.co` という文字列
     - **anon public**（合いカギ）：`eyJ〜〜` で始まるとても長い文字列
       > この「合いカギ」は人に見せても問題ないカギです。安心してコピペしてください。

5. **アプリに住所と合いカギを教える**
   - このフォルダの中の **[`js/config.js`](js/config.js)** というファイルを開きます（メモ帳やテキストエディタでOK）
   - 上のほうにある2行を、さっきコピーした値に **貼り替えます**：
     ```
     supabaseUrl:     'ここに Project URL を貼る',
     supabaseAnonKey: 'ここに anon public を貼る',
     ```
   - 上書き保存すれば準備1は完了です。

---

### 準備2：みんなが開けるように「ネットに公開」する（GitHub Pages）

作ったアプリを、URLでみんなが開けるようにします。GitHubの無料機能を使います。

1. まず、準備1で書き替えたファイルを含めて、変更をGitHubに保存（プッシュ）します。
2. GitHubのこのリポジトリのページを開き、上のほうの **「Settings（設定）」** タブを押します。
3. 左メニューの **「Pages」** を押します。
4. 「Build and deployment」の **Source** を **「Deploy from a branch」** にします。
5. **Branch** を `main`、その隣のフォルダを **`/ (root)`** にして、**「Save」** を押します。
6. 少し待つと（30秒〜数分）、公開用のURLが表示されます。アドレスはこの形になります：
   ```
   https://<あなたのGitHubユーザー名>.github.io/<リポジトリ名>/community-quiz/
   ```
   このリポジトリの場合は：
   **`https://lachimym.github.io/lachi-claude-config/community-quiz/`**
7. このURLをコミュニティのみんなに教えれば完成です！🎉

## 📱 みんなの使い方

| やりたいこと | やり方 |
|---|---|
| **ログイン** | 最初の画面で自分のMisskeyサーバー（例：`misskey.io`）を入れて「ログイン」→ 出てきた画面で「許可」を押す |
| **クイズを解く** | 上の「挑戦」→ 難易度を選ぶ → 答えだと思う選択肢をタップ |
| **結果をシェア** | 正解・不正解の画面で「Misskeyでシェア」→ 投稿画面が文章入りで開くので、そのまま投稿 |
| **問題を作る** | 上の「作問」→ 難易度・問題文・選択肢4つを入れて、正解の左の◯を選んで「登録」 |
| **問題を直す** | 「問題一覧」→ 直したい問題を開く →「この問題を修正」 |
| **コメント・補足を書く** | 問題を開いて、下の入力欄から「コメント」か「補足」を選んで送信 |
| **誰が作ったか見る** | 問題を開くと、下のほうに作った人・直した人・更新日と、これまでの履歴が出ます |

## ✏️ あとから変えたいとき

**[`js/config.js`](js/config.js)** というファイルの中を書き替えるだけです。プログラムの知識はいりません。

- **難易度の名前**（ビギナー/中級/マニアなど）を変えたい
  → `label` の右の文字を書き替える（例：`'ビギナー'` を `'かんたん'` に）
  ※ その左にある `key`（`beginner` など英語の部分）は **変えないで** ください
- **アプリの名前** を変えたい → `appName` の文字を書き替える
- **最初に表示されるMisskeyサーバー** → `defaultMisskeyHost`
- **シェアするときにつくハッシュタグ** → `shareHashtag`

書き替えたら保存して、また準備2と同じようにGitHubに保存（プッシュ）すれば反映されます。

---

## ❓ 困ったときは

- **「Supabaseが未設定です」と黄色い帯が出る**
  → 準備1の手順5（`js/config.js` への貼り替え）がまだか、貼り間違いです。もう一度確認してください。
- **問題が表示されない / 保存できない**
  → 準備1の手順3（`schema.sql` のコピペ実行）ができているか、Supabaseの画面で確認してください。
- **ログインしても戻ってこない**
  → `file://〜` のようにファイルを直接開いていると失敗することがあります。準備2で公開したURL（`https://〜`）から開いてください。

<details>
<summary>ちょっと技術的なメモ（読まなくてOK）</summary>

- ビルドや専用サーバー不要の静的サイト（HTML / CSS / バニラJavaScript）です。
- ログインはMisskeyの **MiAuth**（`write:notes` 権限）をブラウザ内で実行し、トークンは各端末に保存します。
- データ保存は **Supabase**（Postgres）。`@supabase/supabase-js` をCDN経由で読み込みます。
- シェアは `https://<host>/share?text=...` を開く方式なので確実に動作します。
- 現在のデータベースの権限（RLS）は「合いカギを知っていれば読み書きできる」内輪向けの緩い設定です。URLとカギが広く出回らない前提の運用です。より厳密にしたい場合は、Supabase側のポリシー変更で「更新は作成者本人のみ」等にできます。

</details>
